# MVP

## Целевой запуск

Через **8 недель от 2026-04-05** ≈ начало июня 2026.

Прототип проверки договора — задача CTO Димы к концу апреля ([[../articles/meeting-2026-04-15-strategy]]).

## Что в MVP

1. Загрузка **PDF/DOCX**
2. Анализ → выделение рисков с подсветкой
3. Категоризация: **критичные / средние / на заметку**
4. Объяснения «простым языком»
5. Чат по договору
6. Личный кабинет + история проверок
7. Оплата через **ЮKassa** (разовая + подписка)

Все 7 пунктов закрывают функциональные требования из [[../articles/custdev-anna-designer]] — кроме шаблонов договоров.
